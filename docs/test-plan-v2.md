# htmlcli 集成测试计划 v2

> 版本: v2 (2026-04-05)
> 变更: 升级测试协议 — 每个测试 15-20 页，3 轮完整交互（每轮 = 用户可视化编辑 + 自然语言描述）

## 测试方法

### 测试规格
- **每个 deck 15-20 页** — 满足真实 PPT 规模
- **3 轮完整交互** — 每轮包含两步：
  1. **Edit Pass**: 模拟用户在 GrapesJS 可视化编辑器中拖拽/修改（通过 server API）
  2. **Refinement Pass**: 用户用自然语言描述修改意图，AI 读取 changelog + HTML 后智能润色
- **使用图像生成** — 需要图表时调用 `htmlcli asset gen-chart` 或 Python 脚本生成 PNG
- **使用新的 SKILL 规范** — 严格遵循 design-tokens, layout-patterns 等参考文件

### 每轮交互协议

**Round N:**
1. **Edit** (用户可视化编辑):
   - 模拟用户的点击/拖拽/文字编辑行为
   - 通过 `POST /api/save` 提交修改后的 HTML
   - 验证 `.changelog.json` 正确生成

2. **NL Refinement** (用户自然语言描述 + AI 润色):
   - 给出用户的自然语言请求（"帮我把字体调大，增加一页介绍团队"等）
   - AI 读取 HTML + changelog，理解用户意图
   - AI 执行润色/修改，可能包括：调整文字、添加/删除页、重排布局、生成新图表

**总计**: 每个测试 = 3 × (Edit + NL Refinement) = 6 次 AI 操作点

### 评估标准

| 维度 | 通过条件 |
|------|---------|
| 页数规模 | 至少 15 页，每页结构完整（header/body/footer） |
| 设计质量 | 使用 design tokens、16:9 固定画布、不出现 common-mistakes |
| 数据可视化 | 使用 matplotlib 生成 PNG 或 mermaid/Chart.js 客户端渲染 |
| Edit 追踪 | changelog 准确记录所有可视化编辑 |
| NL 理解 | AI 正确理解自然语言意图并执行 |
| 一致性 | 3 轮后整体风格保持一致，无破坏性改动 |
| 文件完整性 | 每轮后 HTML 可正常渲染 |

**基础需求**: Q1 2026 季度业绩汇报, 目标 15-18 页, 财务/技术部联合汇报

## 共享约束

1. **所有测试必须**:
   - 使用 `references/slide-template.md` 的基础模板
   - 遵循 design-tokens.md 的 CSS 变量
   - 至少生成 2 张 matplotlib 图（通过 `htmlcli asset gen-chart` 或 `save_external_image`）
   - 至少一页使用 mermaid 或 Chart.js 客户端渲染

2. **不允许**:
   - 使用占位符（"TODO", "Lorem ipsum"）
   - 硬编码颜色（必须用 CSS 变量）
   - 页数少于 15

3. **每个测试的输出产物**:
   - `test-output/testNN.html` (15-20 页)
   - `test-output/testNN-assets/*.png` (至少 2 张图表)
   - `test-output/testNN.changelog.json` (每轮编辑后更新)
   - `docs/test-results/test-NN-*.md` (测试报告)
